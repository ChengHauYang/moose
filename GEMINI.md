# Project Instructions

To ensure consistent and efficient development, all Gemini CLI sessions MUST follow these foundational steps before starting any task:

1. **Compilation & Build:** Read `CLAUDE.md` to understand the build commands, compilation process, and development environment setup.
2. **Project Context:** Read all other markdown files in the root directory to understand the project structure, history, and specific guides:
    - `README.md`: General overview.
    - `MOOSE_CMAKE_BUILD_GUIDE.md`: Detailed CMake build instructions.
    - `MOOSE_CMAKE_COMPILE_README.md`: Compilation specifics for CMake.
    - `MOOSE_MAKE_BUILD_GUIDE.md`: Detailed Make build instructions.
    - `modules/README.md`: Module descriptions and dependency map.
    - `progress_make2CML.md`: Information regarding the transition or progress of build systems.

These documents contain critical architectural and workflow mandates that take precedence over general defaults.
